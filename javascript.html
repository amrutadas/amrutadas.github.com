The Where Why and What Of JavaScript..


Designing and creating a web page is a truly fun experience. But then comes the most important part  i.e. Making them Dynamic. 


This is where Javascript comes into the picture. Here I am to explain : 'What' is javascript , 'Why' do we use javascript, and 'Where'do we use it.


JavaScript was designed to add interactivity  (i.e. Dynamism ) to HTML pages
It is a scripting language. Its is basically a lightweight programming language. It consists of lines of executable code, just like any other language. 


It is usually embedded into HTML pages. It is an interpreted language(i.e that the script executes without and preliminary complication) . It basically gives a Designer a programming tool. It is a loosely typed language and its syntax is very intuitive and easy.This makes it easier for anyone to learn it and insert small pieces or snippets of code into their HTML page, making their page interactive.

Besides Javascript can react to events : In the above example the event is an onclick event of a button. But there are various other events like when a page has finished loading. On these events Javascript can read and change the content of HTML element.
It is also used to validate data, before submitting to server, thus saving the processor from extra processing.
It can be used to create cookies and to detect the visitor's browser as well.

Now let us consider the benefits and disadvantages of JavaScript :


Benefits 
- associative arrays
- loosely typed variables
- regular expressions
- objects and classes
- highly evolved date, math, and string libraries
- W3C DOM support in the JavaScripts 


Disadvantages 
- Developer depends on the browser support for the JavaScript
 This is indeed the biggest disadvantage of Javascript. If scripting is not enabled on a browser then it will not render anything. Newer versions of javascript have almost done away with this impediment.


In my last blog I mentioned the best practices that one must keep in mind when we design a web page. 
Enlisted below are some of the Good practices that one must keep in mind while incorporating javascript into an HTML page. 


- Separate behavior from content and presentation : Just as we seperate CSS from HTML,  we should sepearte Behavior (Javascript). Just as we link to an external CSS file we should link to external javascript files. It should not be inline.


 The following diagrammatic representation will help you understand better .
 

for eg : Instead of hard coding behavior into content (like onclick, mouseover etc) , behavior should be dynamically added to the elements, classes and unique elements(ID's), using the DOM elements.

- Provide progressive enhancement : Javascript should augment content by adding behavior. Basically content should be useful and usable even without it(or without full javascript support. ). It should allow users to change the content, and not debar them from the content.

- Code for Flexibility : Developers should wisely choose or not the javascript will be triggered by elements, classes or unique element (ids). Markup should be semantic in structure and should be meaningful. 

- Alter content as little as possible : If the behavior (Javascript ) changes just the presentation of the content, then better not to use it. If it changes behavior of content, it should output simple valid HTML .

- Document output, parameters, and dependencies : If the behavior changes the content or outputs additional content, document what the output will look like.Along the same lines, javascript libraries often bundle together numerous functions that control a wide variety of behavior. For each function, document any library dependencies. Knowing library dependencies allows developers to remove portions of the library they won't be using and optimize their application. 

Talking about javascript we come across the term  ' DOM ' several times. Javascript targets the DOM elements. So one need to understand what the DOM elements are.

The Document Object Model (DOM) is a model that describes how all elements in an HTML page, like input fields, images, paragraphs etc. are related to the topmost structure: the document itself. By calling the elements by its proper DOM name, we can act on them. Each HTML document loaded into a browser window becomes a Document object. This Document object provides access to all HTML elements in the page, from within a script. 

For eg:   document.getElementById('elementID').value : This kind of a script fetches the value which is entered (from browser) to the input element with ID= "elementID".
